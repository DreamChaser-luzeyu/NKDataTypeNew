(a)
A circular list may be reversed by reversing the direction of the pointers in each node. For this, we use three pointers to march through the circular list from left to right. current points to the node whose pointer (link) we are about to reverse; prev points to the node on its left; and next points to the node at the right of current. The link in current is changed from next to prev. Then prev, current, and next are advanced one node to the right. The code for the member function to reverse a circular list is given below and in the file icircle.h. 

template<class T>

HCircular<T>& HCircular<T>::Reverse()

{// In-pace reversal of the circular list.

   ChainNode<T> *prev = head,  // previous node

                *current = head->link,

                               // current node

                *next;         // next node      

   while (current != head) {

      // change pointer direction

      next = current->link;

      current->link = prev;



      // move to next node

      prev = current;

      current = next;

      }



   // reverse pointer in head node

   head->link = prev;



   return *this;

}

(b)
The complexity is linear in the length of the circular list.
